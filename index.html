Requirement Gathering & PRD

Before writing a single line of code, collect all requirements using structured questions. Convert them into a Product Requirement Document (PRD) that guides the entire build.

GPT PROMPT — REQUIREMENTS GATHERING
ChatGPT / Claude — Requirement Gathering Prompt

You are a senior software architect. I need you to help me define a complete PRD for my project. Ask me ALL of the following structured questions and wait for my answers before proceeding:

  1. Project Idea: What does this application do? What problem does it solve?
  2. Target Users: Who will use it? (age group, technical level, geography)
  3. Core Features (MVP): List the minimum features needed to launch.
  4. Authentication: Do you need login/signup? Yes or No?
  5. Roles: Do different users have different permissions? (Admin, User, Vendor, etc.)
  6. Platforms: Web app? Mobile app? Or both?
  7. Tech Preferences: Any preferred frameworks or languages?
  8. Scale: Expected users at launch vs 12 months? (Small / Medium / Enterprise)
  9. Integrations: Any 3rd party services? (Stripe, WhatsApp, Google Maps, SMS)
  10. UI Style: Modern SaaS / Marketplace / Dashboard / Minimal / Other?
  11. Database: MongoDB (flexible) or PostgreSQL (relational)?

After I answer, generate a complete PRD document with: Project Overview, User Personas, Feature List (grouped by priority), Module Breakdown, and a 4-phase Development Roadmap.

ℹ️

Pro Tip: Paste this prompt into ChatGPT-4 or Claude. Answer each question thoroughly. The better your answers, the more precise your PRD and feature breakdown will be. Save the PRD as PRD.md in your project root.

02

Tech Stack Selection

Every tool must be chosen deliberately. Below is the recommended production-ready stack with justifications for each choice.

⚛️

Next.js 14 (App Router)

React framework with SSR, SSG, file-based routing and API routes built in.

→ SEO + performance + full-stack in one
🎨

Tailwind CSS

Utility-first CSS framework. Consistent design, zero CSS file bloat, rapid UI development.

→ 10x faster styling than vanilla CSS
🏗️

NestJS (Node.js)

Enterprise-grade Node.js framework. Modular, TypeScript-first, built-in DI container.

→ Scalable + type-safe backend
🗄️

PostgreSQL + Prisma

Relational DB with ACID compliance. Prisma ORM for type-safe database queries.

→ Reliable relations + auto-generated types
🔐

JWT + Passport.js

Stateless auth with access/refresh tokens. Passport strategies for Google OAuth support.

→ Secure, scalable authentication
📱

React Native (Expo)

Cross-platform iOS + Android from one codebase. Expo SDK for camera, push, maps and more.

→ Same JS knowledge, 2 platforms
🔄

Zustand / React Query

Lightweight global state (Zustand) + server state caching and sync (TanStack Query).

→ No Redux boilerplate
🚀

Vercel + Railway

Frontend on Vercel (zero-config Next.js). Backend + DB on Railway (auto scaling).

→ Deploy in minutes
✅

Stack Decision Rule: Match complexity to project size. For a startup MVP, this stack handles up to 100K MAU. For enterprise scale, swap Postgres for CockroachDB and NestJS for microservices.

AUTH MODULE — JWT IMPLEMENTATION
auth/auth.module.tsTYPESCRIPT
import { Module } from '@nestjs/common'; import { JwtModule } from '@nestjs/jwt'; import { PassportModule } from '@nestjs/passport'; import { ConfigModule, ConfigService } from '@nestjs/config'; import { AuthService } from './auth.service'; import { AuthController } from './auth.controller'; import { JwtStrategy } from './strategies/jwt.strategy'; @Module({ imports: [ PassportModule, JwtModule.registerAsync({ imports: [ConfigModule], useFactory: (config: ConfigService) => ({ secret: config.get('JWT_SECRET'), signOptions: { expiresIn: '15m' }, }), inject: [ConfigService], }), ], controllers: [AuthController], providers: [AuthService, JwtStrategy], exports: [AuthService], }) export class AuthModule {}
auth/auth.service.tsTYPESCRIPT
import { Injectable, UnauthorizedException } from '@nestjs/common'; import { JwtService } from '@nestjs/jwt'; import { PrismaService } from '../prisma/prisma.service'; import * as bcrypt from 'bcryptjs'; @Injectable() export class AuthService { constructor( private prisma: PrismaService, private jwt: JwtService, ) {} async register(dto: RegisterDto) { const hash = await bcrypt.hash(dto.password, 12); const user = await this.prisma.user.create({ data: { ...dto, password: hash }, }); return this.signTokens(user.id, user.email, user.role); } async login(dto: LoginDto) { const user = await this.prisma.user.findUnique({ where: { email: dto.email }, }); if (!user) throw new UnauthorizedException('Invalid credentials'); const valid = await bcrypt.compare(dto.password, user.password); if (!valid) throw new UnauthorizedException('Invalid credentials'); return this.signTokens(user.id, user.email, user.role); } private async signTokens(userId: string, email: string, role: string) { const payload = { sub: userId, email, role }; return { accessToken: await this.jwt.signAsync(payload), refreshToken: await this.jwt.signAsync(payload, { expiresIn: '7d' }), }; } }
guards/jwt.guard.ts + roles.guard.tsTYPESCRIPT
// ── JWT Guard (protects routes) ────────────────────────────────── import { AuthGuard } from '@nestjs/passport'; import { Injectable } from '@nestjs/common'; @Injectable() export class JwtAuthGuard extends AuthGuard('jwt') {} // ── Roles Decorator ────────────────────────────────────────────── import { SetMetadata } from '@nestjs/common'; export const Roles = (...roles: string[]) => SetMetadata('roles', roles); // ── Roles Guard (RBAC) ─────────────────────────────────────────── @Injectable() export class RolesGuard implements CanActivate { constructor(private reflector: Reflector) {} canActivate(context: ExecutionContext) { const roles = this.reflector.get<string[]>('roles', context.getHandler()); if (!roles) return true; const req = context.switchToHttp().getRequest(); return roles.includes(req.user.role); } } // ── Usage in Controller ────────────────────────────────────────── @UseGuards(JwtAuthGuard, RolesGuard) @Roles('admin') @Get('admin-only') adminRoute() { return 'Only admins here'; }

DEPLOYMENT CHECKLIST

🔒 Secure all environment variables

Move all secrets to Railway/Vercel env vars. Never commit .env to GitHub. Use strong JWT_SECRET (32+ chars random string).

🗄️ Run production migrations

Run npx prisma migrate deploy (not dev) on your production database. Confirm all tables exist via pgAdmin or Prisma Studio.

🌐 Configure CORS

In NestJS main.ts: app.enableCors({ origin: 'https://your-vercel-url.vercel.app' }). Never use wildcard * in production.

📱 EAS Build for mobile

Update API baseURL in mobile/services/api.ts to production URL. Run eas build --platform all then submit to stores.

✅ Smoke test production

Test register → login → dashboard on production URL. Check browser console for errors. Test on mobile device via Expo Go or production build.
